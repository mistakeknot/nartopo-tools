1.  **Map-Reduce + Semantic Indexing (The Hybrid Index):**
    Instead of running a single RAG query against the *whole* book, run the Map-Reduce pipeline where each sub-agent *first* indexes its 150k chunk locally. It then answers specific structured questions about its chunk. This guarantees no plot beats are missed (because every chunk is queried), but saves tokens because the sub-agent doesn't need to summarize the chunk, it just answers the specific framework questions via its local index.
2.  **Sliding Window Summarization:**
    Instead of Map-Reduce reading discrete chunks in isolation, Agent 1 reads Chunk 1 and outputs a 1000-token summary. Agent 2 receives Chunk 2 *and* the summary from Agent 1. Agent 3 receives Chunk 3 and the summary from Agent 2. This guarantees perfect continuity and pacing awareness across the entire book, which Map-Reduce sometimes struggles with at the chunk boundaries.
3.  **Intermediate Structural Representation (JSON-Lines):**
    Instead of asking sub-agents for bullet points (which are unstructured and variable length), force the sub-agents to output a strict JSONL file of *events*, with metadata like `{"type": "action|dialogue|bureaucracy", "timestamp_or_page": X, "characters_involved": [], "summary": "..."}`. The primary agent then just reads this highly compressed, standardized JSON timeline to evaluate pacing (by looking at the ratio of "action" vs "bureaucracy" events) rather than reading raw text.
